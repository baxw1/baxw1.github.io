如何免费搭建自己的个人博客

看到朋友做了自己的个人博客，羡慕了蛮久了，拖延到现在才动手试一试；

服务器啥的太难了，百度了个超级简单的办法

记录一下，方面自己调试

本篇也适合电脑小白学习，超级简单！

附上我学习的博主的链接：

一个更快速的教学

更适合新手的手把手教学链接

Step1：下载安装node

百度搜一下咋下载吧，我给忘了，等我想起来就把下载链接放在这里

安装成功后输入 node -v，如果输出类似 v8.11.4 ，则说明你下载安装成功

Step2：安装hexo

在cmd下输入： npm install hexo-cli -g 来安装hexo

Step3：初始化博客文件

cmd下输入hexo init myblog

如果你是Windows用户，你可以在C盘用户文件夹下找到新生成的网站根目录：myblog

随后，在cmd下进入根目录： cd myblog，安装npm install

输入 hexo clean 可以清除缓存（刷新）

输入 hexo g 生成静态页面

输入 hexo s 运行你的blog

Step4：查看是否初步配置成功

打开浏览器输入 localhost:4040 ，即可看到初步效果

如何让别人看到你的博客

Step1：创建一个GitHub账号

不多赘述，进入Github网页注册登录即可

记得取一个喜欢的账号名，这很重要

Step2：创建自己的Github pages仓库

点击绿色的 New

在输入框中填入 baxw1.github.io

注意要把baxw1替换成自己的账户名

且！！！一定要与前面Owner的名字完全一致

勾选 Public 和 Add a README file ，随后点击 Create repository 即可完成创建

Step3：将自己的本地网页文件传到Github上

为了让你只能本地访问的博客网页可以被所有人看到，你需要将文件部署到仓库上

首先，安装 hexo-deployer-git，进入你的网页根目录下，输入：

npm install hexo-deployer-git --save

用记事本打开根目录下的 _config.yml 文件，翻到最下面，修改这里的配置：

deploy:

type: git

repo: git@github.com:baxw1/baxw1.github.io.git

branch: main

同样的，把baxw1替换成自己的Github用户名

注意语法！每一个冒号后面必须带一个空格！

​

回到cmd中运行 hexo deploy -d (依然要在根目录下运行)

运行完后，在你的浏览器中输入 baxw1.github.io 就可以看到自己的个人博客啦

根据我的经验，这一步往往会出现许多问题，我在这里mark一下

  1. 第一次运行一般会让你登录认证，完成登录认证即可

  2. 另外，在上传到GitHub时，最好使用VPN，因为国内连接Github并不稳定，可能会在hexo d后出线timeout问题

  3. 如果遇到“Please make sure you have the correct access rights and the repository exists.”问题

    1. 进入到需要提交的文件夹底下(在这里就是我们的网站根目录)，输入:(引号内换成你自己的账号和邮件名)

      git config --global user.name "yourname"

      git config --global user.email“your@email.com"

    2. 删除.ssh文件夹下的known_hosts文件(应该位于C:\Users\用户名.ssh)

    3. 输入指令 ssh-keygen -t rsa -C "your@email.com"

    4. 出现”Generating public/private rsa key pair. Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):”后，按下回车，系统就会在.ssh文件夹下生成文件id_rsa和id_rsa.pub，用记事本打开id_rsa.pub，复制其中的内容

    5. 进入Github网页，登录你的账户：设置->SSH and GPG Keys->New SSH Key，将key复制粘贴进去，点击 Add SSH key

    6. 输入命令 ssh -T git@github.com ，确认输入yes ，回车

    7. 再进行 hexo deploy -d ，即可

  4. 每次hexo d后，立马进入baxw1.github.io网站可能还未更新，可以稍等一会儿，或者清一下缓存

如何发布你的文章

在文件夹下输入hexo new "文章名字" ，这时你就可以在根目录下的source -> _post文件夹下看到新生成的 文章名字.md文件，用markdown语法编写你的文件就可以了

推荐的markdown编辑器：Typora

markdown语法教程：Markdown官方教程

一个后面发现还满全面的教程：Typora 完全使用详解 - 少数派 (sspai.com)

Markdown在线编辑器：在线编辑器

编辑好你的文章后，保存，再刷新本地http://localhost:4000， 检查文章是否正确

确认好文章后，再hexo deploy -d就可以啦
